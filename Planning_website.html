Brief

I am making a pasta bar ordering system. It is based from the employes side and they are putting in each order that the customer that is meant to be over the phone is ordering. There needs to be able to add and remove items from the list and get customer details from them, also asking if they want pickup or delivery.

Relevant implications

Usability

Usability is ensuring that the user can understand and use the program. The user should not be confused by the language and there isn’t any ‘tech’ language that the user would not be able to understand. To give appropriate feedback to the user when it is needed and prevent users being confused about what they need to do at each step. This means I need to ensure that there are proper instructions for the user and if they need feedback that it is helpful for them to understand how the ordering system program works. It needs to have some validation in place for error prevention and be able to communicate with the user when they have made an error, what they have done wrong, and how to fix it. Each option should be easy to understand for the user e.g. enter in a for add item to the customer order list.

Functionality

The purpose of this relevant implication is making sure that the code actually works. That I have all my validation in where it is needed to be and it actually works and is function for an online ordering system. In order to do this you have to make sure that my the ordering system does not crash. I need to make sure that the program has no bugs or faults so it runs smoothly and doesn’t crash. It did not do incorrect maths when adding up the subtotals or overall total of the order. All items added to the separate lists e.g. customer order list will resets if the user requests to run again; all lists need to be emptied if user wants to order again.

Sustaibility and future proofing

Sustainability and future proofing is guaranteeing that I have properly commented and doctyped my code. This is so I could go back and look at previous code I have written and understand what is meant to happen and what I was writing. As well as that it is also if someone else were to look at my code they would understand how it works andbe able to give to another developer and them understand it quickly. This means that everything should appropriately commented before important functions/code and that every function is doctyped properly. This relevant implication is also for if I could add new features and/or functions to the code easily and I would not have to alter previous code at all or too much to add in new functionality for this code.
