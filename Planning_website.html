Sprint One

Brief

I am making a pasta bar ordering system. It is based from the employes side and they are putting in each order that the customer that is meant to be over the phone is ordering. Sprint one is a simple design where you have two options. Those two are that you are able to print the list of just the pastas and then the second option of quitting the program.

Planning for sprint one

Sprint one boards

Reflection

This sprint was very basic code and it just has would you like to review the list or would you like to stop the program. There are a lot of improvements that could be made to my code. Going forward I should focus on the styling a little bit more because when it prints out the pasta list it prints it not in a nice way. There isn't any features to add anything to the customer order in the future it should ask for the order and print the name of the resturant. It is very unformatted this version so I want the menu list to be more formatted and look nicer.

Reflection

This sprint is still very basic and ther isn't any validation In this sprint I added a feature where the user can add items to the order and added the function to the main list options.
